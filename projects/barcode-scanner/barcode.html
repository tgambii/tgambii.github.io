Barcode Scanner
& SAP Integration

A Python desktop application built to streamline parts tracking in the Curtiss-Wright hardware lab — scanning physical barcodes, resolving SAP part numbers, displaying part images, and exporting build reports to Excel.

Company
Curtiss-Wright · Tewksbury, MA
Role
Mechanical Design Engineer
Type
Internal tooling (Python)
Status
⚠ Awaiting API approval
⚠️
Project status — pending IT approval The application is fully built and functional using a manual parts list. Full deployment was blocked because the SAP BASIS team did not approve API endpoint access to Material Master data. The tool is production-ready and could be activated with a single endpoint URL and credentials.
Overview

The problem it solved.

Tracking which parts are pulled from the hardware lab for each build required manually cross-referencing a physical parts list, looking up SAP material numbers, and entering quantities into a spreadsheet by hand.

It was slow, error-prone, and entirely dependent on an engineer knowing the SAP part numbering system by memory. The goal was to replace that process with a barcode scanner — scan the label on a bin, get the SAP part number instantly, see a photo of the part for visual confirmation, enter a quantity, and export a clean report at end of build.

📷
Barcode resolution
USB scanner input matched against a pre-built EAN-13 lookup table to return the SAP part number instantly.
🖼️
Part image display
On match, the app loads SAP_PN.png from a local folder for visual confirmation before quantity is committed.
📊
Excel export
Quantities written to a formatted .xlsx file named after the build number, ready for SAP entry.
🔗
SAP API hook
Architecture supports live Material Master fetching via SAP OData REST API with secure credential management.

Technical breakdown

How it works.

Parts at Curtiss-Wright follow a consistent naming convention: TCG-HL-{PREFIX}-{NUMBER}. The barcode generation algorithm converts the letter prefix to a numeric sequence, pads the result to 12 digits, and calculates a valid EAN-13 check digit — making the output scannable by any standard USB barcode reader.

# EAN-13 generation — convert SAP P/N to a scannable barcode def generate_barcode(entry, letter_map): prefix, number = entry.split("-")[2], entry.split("-")[3] barcode = "0000" + replace_letters(prefix, letter_map) + number digits = [int(d) for d in barcode] total_sum = sum(digits[i] if i % 2 == 0 else digits[i] * 3 for i in range(12)) check = str((10 - (total_sum % 10)) % 10) return barcode + check

Development process

How it was built.

1
Step 01
Command-line prototype
Started as a pure terminal application. Validated the EAN-13 algorithm against real part numbers before building any UI.
2
Step 02
EAN-13 barcode map
Built the letter-to-digit encoding system and check digit algorithm to convert proprietary part numbers into valid EAN-13 barcodes readable by any off-the-shelf USB scanner.
3
Step 03
SAP API integration
Designed the API client to pull live Material Master data from SAP ERP using OData REST with secure credential management via environment variables. Hit blocker — IT approval not granted.
4
Step 04
Desktop GUI
Built the full tkinter interface — part image display, quantity confirmation, live scanned-items table, and one-click Excel export named after the build number.
5
Step 05
Manual fallback & packaging
Added a manual parts list fallback so the tool is fully usable without the API. Explored PyInstaller packaging for distribution to Windows lab machines without Python installed.
⛔
Deployment blocker The SAP BASIS team did not provide an approved OData API endpoint for the Material Master service. The application is production-ready and the API client is fully implemented — it requires only a valid endpoint URL and credentials to go live. The project remains in manual-list mode pending IT approval.
